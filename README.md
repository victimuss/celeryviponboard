# 🚀 VIP Onboarding Pipeline | FastAPI + Celery + Redis

Этот репозиторий — шаблон production-ready микросервиса. Он демонстрирует, как правильно обрабатывать тяжелые и нестабильные фоновые задачи, чтобы ваш API всегда отвечал пользователям за миллисекунды.

## 🧠 Суть проекта
Когда пользователь покупает VIP-статус, система должна выполнить ряд сложных действий: сгенерировать PDF-карту, загрузить её в облако (AWS S3) и отправить email со ссылкой. Если делать это синхронно в основном потоке, сервер "зависнет" на 8-10 секунд. 

В этом проекте FastAPI выступает исключительно как менеджер — он моментально отвечает пользователю, а всю тяжелую работу передает распределенной очереди задач Celery.

## ✨ Ключевые архитектурные решения
* **⛓️ Celery Chains (Конвейеры):** Задачи не монолитны. Они разбиты на микро-функции, которые передают данные друг другу по эстафете (Генерация ➡️ Загрузка ➡️ Email).
* **🛡️ Smart Retries (Умные повторы):** Реализована имитация обрыва связи при обращении к внешнему облачному хранилищу. Воркер перехватывает ошибку сети, не "убивает" задачу, а делает `self.retry()` с задержкой и пробивается со второй попытки.
* **⏰ Celery Beat (Планировщик):** Поднят фоновый крон-процесс, который каждую минуту начисляет VIP-пользователям бонусные баллы абсолютно независимо от основного API.

## 🛠 Технологический стек
* **Backend:** FastAPI (Python 3)
* **Task Queue:** Celery
* **Message Broker:** Redis (Upstash)

---

## ⚙️ Структура проекта
* `main.py` — Эндпоинты FastAPI и инициализация цепочки задач.
* `worker.py` — Настройки Celery, Beat и сама логика фоновых воркеров.
* `.env` — Файл переменных окружения (не трекается в Git).
* `requirements.txt` — Зависимости проекта.

## 🚀 Как запустить локально

### 1. Клонирование и зависимости
```bash
git clone [https://github.com/victimuss/celeryviponboard.git](https://github.com/victimuss/celeryviponboard.git)
cd НАЗВАНИЕ_РЕПО
pip install -r requirements.txt
```
### 2. Настройка переменных
```
REDIS_URL=rediss://default:ВАШ_ПАРОЛЬ@ВАШ_ХОСТ.upstash.io:6379
```
### 3. Запуск микросервисов
Для работы всей архитектуры потребуется открыть 3 отдельных терминала. Убедитесь, что в каждом из них активировано виртуальное окружение (venv).

🖥 Терминал 1: Запуск API (FastAPI)

```bash
uvicorn main:app --reload
```
👷‍♂️ Терминал 2: Запуск Воркера (Celery Worker)
(Примечание: флаг --pool=solo решает проблему многопроцессорности на Windows)

```bash
celery -A worker worker --loglevel=info --pool=solo
```
⏰ Терминал 3: Запуск Планировщика (Celery Beat)

```bash
celery -A worker beat --loglevel=info
```
